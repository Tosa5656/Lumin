# Contribution to Lumin

We welcome your interest in participating in the development of the Lumin programming language!  Contributors who adhere to the rules below help create an open and productive community.

## How can I contribute?

There are many ways to contribute to Lumin:

* **Bug Reports:** If you find a bug in Lumin, please report it by opening an Issue on GitHub.  Be sure to specify:
    * The Lumin version (if applicable).
    * The operating system.
    * Detailed description of the error and steps to reproduce it.
    * Sample code demonstrating the problem.
* **Suggestions for improvement (Feature Requests):** If you have an idea to improve Lumin or add new functionality, please suggest it by opening an issue on GitHub.  Describe your idea in as much detail as possible and explain why it is important.
* **Improved documentation:** Documentation is a key part of any project.  You can help improve the Lumin documentation by correcting errors, adding code examples, or making it clearer.
* **Code Development:** If you would like to contribute to the Lumin codebase, please review our development principles (see below).
* **Testing:** Help test new features and bug fixes to make sure they work correctly.

## Principles of development

* **Follow the code style:** Lumin has its own code style.  Please follow it when writing the code.  The details will be described in a separate file. [CODE_STYLE.md](https://github.com/Tosa5656/Lumin/CODE_STYLE.md) (in development).
* **Write tests:** Each new feature and bug fix should be accompanied by tests.
* **Be concise and clear:** Your code should be easy to read and understandable for other developers.
* **Break big changes into small ones:** Break down complex tasks into smaller ones to make them easier to understand and verify.
* **Leave comments:** Explain complex parts of the code using comments.

## The contribution process

1. **Create an issue:** Before you start working on any change, create an issue on GitHub to discuss it with the Lumin development team.  This will help to avoid duplication of work and ensure that your change meets the project objectives.
2. **Make a fork of the repository:** Create a fork of the Lumin repository on your GitHub account.
3. **Create a branch:** Create a new branch in your fork for your work.  Use an informative name for the branch (for example, `fix-bug-in-parser' or `add-new-feature').
4. **Make changes:** Make any necessary changes to the code, documentation, or tests.
5. **Commit the changes:** Record your changes with an informative message describing the nature of the changes.  Use the prefix `fix:` for bug fixes and `feat:` for adding new features.
6. **Send a pull request:** Send a pull request (PR) from your branch to the main Lumin repository.  Describe your changes in PR in as much detail as possible.
7. **Wait for the review:** Your PR will be reviewed by the Lumin development team.  They may ask you to make changes or answer questions.
8. **Make the necessary changes:** Make the necessary changes according to the reviews on your PR.
9. **PR will be accepted (merged):** After your PR is approved, it will be accepted into the main Lumin repository.

## Code of Conduct

We expect all members of the Lumin community to behave respectfully and constructively.  Insults, discrimination, trolling and other forms of unacceptable behavior are not allowed.  Detailed rules of conduct will be set out in the file [CODE_OF_CONDUCT.md](https://github.com/Tosa5656/Lumin/CODE_OF_CONDUCT.md) (in development).

## Gratitude

We are grateful to everyone who contributes to Lumin!  Your contribution helps to make Lumin a better place for everyone.
