# Стиль Кода Lumin
Этот документ описывает стиль кодирования, которому следует придерживаться при написании кода на языке Lumin.  Соблюдение этих правил поможет сделать код более читаемым, понятным и согласованным.

## I. Общие принципы
*   **Читаемость превыше всего:**  Код должен быть легко читаемым и понятным для других разработчиков.
*   **Согласованность:**  Придерживайтесь единого стиля кодирования во всем проекте.
*   **Краткость:**  Пишите код максимально кратко и лаконично, но не в ущерб читаемости.

## II. Форматирование
### Отступы
*   Используйте табуляцию для отступов.
    ```cpp
    if (condition) 
    {
        statement;
    }
    ```

### Длина строки
*   Старайтесь не превышать длину строки в **120 символов**.  Если строка получается длинной, разбейте ее на несколько строк.
    ```cpp
    // Пример разбиения длинной строки
    string longText = "long text..."
                    "continuation of long text.";
    ```

### Пробелы
*   **Вокруг операторов:** Используйте пробелы вокруг операторов присваивания, сравнения, арифметических и логических операторов.
    ```cpp
    int x = 5 + 3;
    if (x == 8) { ... }
    ```
*   **После запятых:**  Используйте пробел после запятой в списках аргументов, массивах и т.д.
    ```cpp
    int array = {1, 2, 3, 4};
    function(arg1, arg2, arg3);
    ```
*   **Вокруг фигурных скобок:**  Ставьте пробел перед открывающей фигурной скобкой в определениях функций, классов, `if`, `for`, `while` и т.д.
    ```cpp
    if (condition)
    {
        // Code block
    }
    ```
*   **Внутри круглых скобок:**  Не ставьте пробелы внутри круглых скобок, окружающих выражения.
    ```cpp
    int result = (5 + 3) * 2; // Правильно
    int result = ( 5 + 3 ) * 2; // Неправильно
    ```

### Переводы строк
*   **После каждого оператора:** Каждый оператор должен заканчиваться переводом строки.
    ```cpp
    int x = 5;
    int y = 3;
    ```
*   **Между блоками кода:**  Оставляйте пустую строку между логически разными блоками кода.
    ```cpp
    // First block
    int x = 5;

    // Second block
    int y = 3;
    ```
*   **После определения функции/класса:**  Оставляйте пустую строку после определения функции или класса.
    ```cpp
    function calculateSum(int a, int b) {
        return a + b;
    }

    // Пустая строка
    int result = calculateSum(5, 3);
    ```

## III. Именование
### Имена переменных
*   Используйте **camelCase** для имен переменных.
    ```cpp
    int myVariable;
    string userName;
    ```
*   Имена переменных должны быть описательными и отражать назначение переменной.
    ```cpp
    int numberOfStudents; // Хорошо
    int n; // Плохо (непонятно, что значит n)
    ```
*   Короткие имена переменных (например, `i`, `j`, `k`) допускаются только в циклах и коротких блоках кода.
    ```cpp
    for (int i = 0; i < array.length(); i++) {
        // Use i as index
    }
    ```

### Имена функций
*   Используйте **camelCase** для имен функций.
    ```cpp
    function calculateArea();
    function getUserName();
    ```
*   Имена функций должны быть глаголами или глагольными фразами, описывающими действие, которое выполняет функция (**опцианально**).
    ```cpp
    function calculateSum(int a, int b); // Хорошо
    function sum(int a, int b); // Плохо (непонятно, что функция делает)
    ```

### Имена классов и структур
*   Используйте **PascalCase** (верхний CamelCase) для имен классов и структур.
    ```cpp
    class MyClass { ... }
    struct UserData { ... }
    ```

### Имена констант
*   Используйте **UPPER_SNAKE_CASE** для имен констант.
    ```cpp
    const int MAX_VALUE = 100;
    const string DEFAULT_USERNAME = "guest";
    ```

### Имена модулей и пространств имен
*   Используйте **PascalCase** для имен модулей и пространств имен.
    ```cpp
    namespace MyModule { ... }
    namespace UserAuthentication { ... }
    ```

## IV. Комментарии
*   **Пишите комментарии:** Комментируйте свой код, чтобы объяснить его логику и назначение.
*   **Используйте JSDoc-подобные комментарии для документирования API:**
    ```cpp
    /**
     * @param {int} a The first number.
     * @param {int} b The second number.
     * @returns {int} The sum of a and b.
     */
    function calculateSum(int a, int b) {
        return a + b;
    }
    ```
*   **Обновляйте комментарии:**  Убедитесь, что ваши комментарии всегда актуальны и соответствуют коду.
*   **Избегайте очевидных комментариев:**  Не пишите комментарии, которые просто перефразируют код.
    ```cpp
    int x = 5; // Присваиваем x значение 5 (Плохой пример)
    ```
    Вместо этого, комментируйте сложные или неочевидные части кода.
    ```cpp
    // Используем алгоритм быстрого возведения в степень для оптимизации вычислений
    int result = fastPower(base, exponent);
    ```

## V. Конструкции языка
### Условные операторы
*   Всегда используйте фигурные скобки для `if` и `else` блоков, даже если они содержат только один оператор.
    ```cpp
    if (condition) 
    {
        statement;
    } 
    else 
    {
        statement;
    }
    ```
    Это помогает избежать ошибок и делает код более читаемым.

### Циклы
*   Всегда используйте фигурные скобки для `for` и `while` блоков, даже если они содержат только один оператор.
    ```cpp
    for (int i = 0; i < 10; i++) 
    {
        statement;
    }
    ```

### Функции
*   Разделяйте функции на небольшие, логически обособленные блоки.
*   Функция должна выполнять только одну задачу.
*   Используйте осмысленные имена аргументов.

### Классы и структуры
*   Используйте классы для представления сложных объектов с состоянием и поведением.
*   Используйте структуры для представления простых данных.
*   Определяйте четкий интерфейс для классов и структур.
*   Используйте инкапсуляцию для защиты внутреннего состояния объектов.

### Исключения
*   Используйте исключения для обработки ошибок и нештатных ситуаций.
*   Бросайте исключения, когда функция не может выполнить свою задачу.
*   Ловите исключения только тогда, когда вы можете их обработать.
*   Не используйте исключения для управления потоком выполнения программы.

## VI. Примеры Хорошего и Плохого Кода
### Пример 1: Именование переменных
```cpp
// Плохо
int a = 5;
int b = 10;
int c = a + b;

// Хорошо
int firstNumber = 5;
int secondNumber = 10;
int sum = firstNumber + secondNumber;
```

### Пример 2: Отступы и пробелы
```cpp
// Плохо
if(condition)
{
  statement;
}

// Хорошо
if (condition) 
{
    statement;
}
```

### Пример 3: Длина строки
```cpp
// Плохо
string veryLongString = "This is a very long string that exceeds the recommended line length and makes the code difficult to read.";

// Хорошо
string veryLongString = "This is a very long string that exceeds the recommended "
                        "line length and makes the code difficult to read.";
```

### Пример 4: Комментарии
```cpp
// Плохо
int x = 5; // Присваиваем x значение 5

// Хорошо
// Инициализируем переменную x начальным значением 5
int x = 5;
```

### Пример 5: Функции
```cpp
// Плохо
function doEverything(int a, int b, string operation) {
    if (operation == "add") 
    {
        return a + b;
    } 
    else if (operation == "subtract") 
    {
        return a - b;
    } 
    else 
    {
        return 0;
    }
}

// Хорошо
function add(int a, int b) {
    return a + b;
}

function subtract(int a, int b) {
    return a - b;
}
```
