# Вклад в Lumin
Мы рады вашему интересу к участию в разработке языка программирования Lumin!  Вкладчики, придерживающиеся правил ниже, помогают создать открытое и продуктивное сообщество.

## Как я могу внести свой вклад?
Есть много способов внести свой вклад в Lumin:

*   **Сообщения об ошибках (Bug Reports):** Если вы нашли ошибку в Lumin, пожалуйста, сообщите о ней, открыв Issue на GitHub.  Обязательно укажите:
    *   Версию Lumin (если применимо).
    *   Операционную систему.
    *   Подробное описание ошибки и шаги для ее воспроизведения.
    *   Пример кода, демонстрирующий проблему.
*   **Предложения по улучшению (Feature Requests):** Если у вас есть идея по улучшению Lumin или добавлению новой функциональности, пожалуйста, предложите ее, открыв issue на GitHub.  Опишите свою идею как можно подробнее и объясните, почему она важна.
*   **Улучшение документации:** Документация - это ключевая часть любого проекта.  Вы можете помочь улучшить документацию Lumin, исправив ошибки, добавив примеры кода или сделав ее более понятной.
*   **Разработка кода:** Если вы хотите внести свой вклад в кодовую базу Lumin, пожалуйста, ознакомьтесь с нашими принципами разработки (см. ниже).
*   **Тестирование:** Помогите протестировать новые функции и исправления ошибок, чтобы убедиться, что они работают правильно.

## Принципы разработки
*   **Следуйте стилю кода:**  Lumin имеет свой собственный стиль кода.  Пожалуйста, следуйте ему при написании кода.  Подробности будут описаны в отдельном файле [CODE_STYLE.md](https://github.com/Tosa5656/Lumin/blob/master/docs/ru/CODE_STYLE.md) (в разработке).
*   **Пишите тесты:**  Каждый новый функционал и исправление ошибок должны сопровождаться тестами.
*   **Будьте краткими и понятными:**  Ваш код должен быть легко читаемым и понятным для других разработчиков.
*   **Разбивайте большие изменения на небольшие:**  Разбивайте сложные задачи на более мелкие, чтобы их было легче понять и проверить.
*   **Оставляйте комментарии:**  Объясняйте сложные части кода с помощью комментариев.

## Процесс внесения вклада
1.  **Создайте issue:**  Перед тем, как начать работу над каким-либо изменением, создайте issue на GitHub, чтобы обсудить его с командой разработчиков Lumin.  Это поможет избежать дублирования работы и убедиться, что ваше изменение соответствует целям проекта.
2.  **Сделайте fork репозитория:**  Создайте fork репозитория Lumin на свой аккаунт GitHub.
3.  **Создайте ветку (branch):**  Создайте новую ветку в своем fork для вашей работы.  Используйте информативное имя для ветки (например, `fix-bug-in-parser` или `add-new-feature`).
4.  **Внесите изменения:**  Внесите необходимые изменения в код, документацию или тесты.
5.  **Зафиксируйте изменения (commit):**  Зафиксируйте свои изменения с информативным сообщением, описывающим суть изменений.  Используйте префикс `fix:` для исправлений ошибок и `feat:` для добавления новых функций.
6.  **Отправьте pull request:**  Отправьте pull request (PR) из своей ветки в основной репозиторий Lumin.  Опишите свои изменения в PR как можно подробнее.
7.  **Дождитесь ревью:**  Ваш PR будет рассмотрен командой разработчиков Lumin.  Они могут попросить вас внести изменения или ответить на вопросы.
8.  **Внесите необходимые изменения:**  Внесите необходимые изменения в соответствии с отзывами на ваш PR.
9.  **PR будет принят (merged):**  После того, как ваш PR будет одобрен, он будет принят в основной репозиторий Lumin.

## Code of Conduct
Мы ожидаем, что все участники сообщества Lumin будут вести себя уважительно и конструктивно.  Не допускаются оскорбления, дискриминация, троллинг и другие формы неприемлемого поведения.  Подробные правила поведения будут изложены в файле [CODE_OF_CONDUCT.md](https://github.com/Tosa5656/Lumin/blob/master/docs/ru/CODE_OF_CONDUCT.md) (в разработке).

## Благодарность
Мы благодарны всем, кто вносит свой вклад в Lumin!  Ваш вклад помогает сделать Lumin лучше для всех.
